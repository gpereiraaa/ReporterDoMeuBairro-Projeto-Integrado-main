# Requisitos Funcionais
| ID  | Nome                                                         | Descrição                                                                                                                                                                                                 |
|-----|--------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1.0 | Registrar novas ocorrências                                  | O sistema deve permitir o cadastro de novas ocorrências.                                                                                                                                                  |
| 1.1 | Descrição de ocorrências                                     | Na descrição deve constar: - A localização exata do problema, inserindo cep e complemento. Ex: “00000-000, casa 12”. - Anexar fotos para ilustrar o contexto. - Informar dados de contato (opcional, mas incentivado para facilitar o acompanhamento). |
| 1.2 | Visualizar detalhes de cada ocorrência                       | O sistema deve permitir visualizar os detalhes de cada ocorrência, contendo descrição, localização, fotos/vídeos (se houver) e histórico de atualizações.                                                |
| 1.3 | Visualizar ocorrências                                       | O sistema deve permitir consultar todas as ocorrências registradas no mapa, exibindo-as de forma geolocalizada, com opção de aplicar filtros.                                                            |
| 1.4 | Filtrar as ocorrências por status                            | O cidadão poderá fazer um filtro de ocorrências com status (pendente, em andamento, resolvido), também podendo filtrar por tipo e data da ocorrência.                                                   |
| 1.5 | Selecionar a categoria do problema                           | O sistema deve permitir selecionar a categoria do problema quando fazer uma nova ocorrência, ex: buraco na rua, iluminação pública, árvore caída, vazamento de água etc.                                  |
| 1.6 | Acompanhar o status das ocorrências                          | O sistema deve permitir ao usuário acompanhar o status das suas ocorrências, visualizando o status em tempo real.                                                                                        |
| 1.7 | Receber notificações sobre atualizações no status das ocorrências | O sistema deve enviar notificações a cada atualização no status de suas ocorrências cadastradas.                                                                                                          |
| 1.8 | Permitir que usuários comentem em ocorrências                | O sistema deve permitir que usuários comentem nas ocorrências públicas, a fim de fornecer mais informações ou atualizações.                                                                               |
| 1.9 | Integrar um sistema de avaliação da resolução das ocorrências | O sistema deve permitir que o cidadão avalie a resolução do problema após concluída.                                                                                                                       |
| 2.0 | Criar um painel administrativo para a gestão das ocorrências pelos órgãos responsáveis | Necessário desenvolver um painel administrativo para que os órgãos gestores possam acompanhar e atualizar o status das ocorrências.                                                                       |
